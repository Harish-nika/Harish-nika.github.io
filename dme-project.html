Data Mining Engine (DME)

SDE Trainee (6-Month Probation) | FactEntry Data Solutions

During my 6-month probation at FactEntry Data Solutions (A SIX Company) as an SDE Trainee, I developed and deployed this massive, high-performance, containerized data extraction platform into production. It serves as an enterprise-grade solution for financial instrument analysis across Corporate and Municipal bond sources.

Core Technical Achievements

  • Process Scaling & GIL Bypass: To fully utilize an 80+ core Xeon server, I engineered the system to bypass Python's Global Interpreter Lock by spawning separate processes via `ProcessPoolExecutor`, enabling it to process 80+ ISINs simultaneously.
  • Role-Based CPU Tiering: Implemented job-level semaphores to cap batches (Low: 5 batches, Medium: 8 batches, High: 16 batches) allocating exact CPU fractions dynamically per user.
  • Microservices Architecture: Built heavily decoupled services using FastAPI, Python, PostgreSQL, and Redis across Corporate, Municipal, Validation, and Document Processing domains.
  • Job Life-cycle: Uses short terminatable batches with direct `/cancel-job` signals and persistent flag restoration for accurate pausing/resuming.
  • SMB Mount Data Infrastructure: Maps massive SMB network shares directly into container file systems via Kubernetes CSI plugins, avoiding HTTP transfer penalties for gigabytes of PDF documents.
